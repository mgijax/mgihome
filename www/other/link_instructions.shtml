How to Create WWW Links to MGI  

Due to MGI's implementation of accession ID handling, we are providing this page of instructions for creating URL links to MGI.Note: We strongly recommend using the new URL for linking to MGI. Other URLs may not be supported in the future.

New URL and Syntax

We have provided a new CGI program for linking to MGI via accession ID:
    http://www.informatics.jax.org/searches/accession_report.cgi?id=xxxx
where xxxx is a comma-separated list of Accession IDs.

The result is one or more objects from MGI associated with the given IDs.

MGI stores objects with accession IDs from many sources. Some examples of valid Accession IDs are

  • MGI:12345 (MGI Accession ID)
  • J:12345 (J Number)
  • GDB:12345 (GDB Accession ID)
  • K02800 (GenBank Sequence ID)
  • GO:0030264 (GO Accession ID)
There are eleven types of objects supported by this URL:
  • References
  • Segments
  • Mouse Markers
  • Alleles
  • Non-Mouse Markers (Homologies)
  • Experiments
  • Expressions Assays
  • Images
  • Antibodies
  • Antigens
  • Gene Ontology (GO) Terms

Additional linking options are available upon request.

What the CGI Will Return

If the search expression matches a single reference, segment, marker or experiment, a detailed description will be displayed. If the search expression matches a non-mouse marker (such as a human gene), a list of homologies will be displayed for that marker. If the search expression matches more than one object, a summary screen will be displayed which lists each match along with its MGI accession ID, Type, Subtype, and a short description.

Data Dumps for Obtaining Accession IDs

All data dumps are tab-delimited and generated nightly. The MGI Accession ID is the first column in each report. These files are available upon request. If interested, please contact MGI User Support.
