MGI Release 2.97 Announcement - November 5, 2003  

MGI is pleased to announce its 2.97 release. The primary features of the release are enhancements to our Gene Expression Database (GXD).

  • Gene Expression Data Expanded Query Form
    This new form lets you search for genes expressed in some anatomical structures and/or developmental stages but not in others.

  • Anatomical Dictionary Browser Term Detail
    This page now provides a link to the full anatomical hierarchy of the Mouse Anatomical Dictionary. You can scroll the resulting display and see immediately where your search term falls by stage and structure in the Dictionary. Click here for a sample search on the term heart.

  • Anatomical Dictionary Browser Query Refiner
    This new form lets you narrow your searches by entering additional parameters, based upon expression results from the standard Mouse Anatomical Dictionary Browser form. Access to the Query Refiner is from an Anatomical Browser Term Detail page.

  • Gene Expression Literature Index Query Form
    A redesign of the summary for these queries allows you to easily determine which references are of most interest. A table summarizes query matches by age and assay type. A number indicates whether or not the paper exists in the detailed data portion of the database and links to the matching index results in each reference. Type a gene name in the Gene Symbol/Name box on the Query Form and click Search to see the new summary.

  • in situ reporter (knock in)
    We added this assay type to the Gene Expression Data Query Form and the Gene Expression Literature Index.

  • Mutant specimens
    You can now query for expression data obtained from mutant specimens using the Mutated Gene field on both the Gene Expression Data and the Gene Expression Data Expanded query forms.

  • Query summaries
    Most of our query summary reports now feature a new section at the top of the page. A box lists the query report title, the fields you selected on the form, your sorting and limiting options, and all the values that your search returns. In general, fields are listed in the same order as they appear on the query form.
